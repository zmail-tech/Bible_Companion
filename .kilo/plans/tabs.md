# Tabbed Passage Reader

## Overview

Add a tab bar above the split-container that lets users open multiple Bible passages simultaneously. Each tab tracks its own book/chapter state and has its own AI commentary panel.

## Changes

### 1. `js/bible.js` ‚Äî Refactor to support per-tab state

Currently `bible.js` uses global `currentBook` / `currentChapter` state. Refactor to create a `TabState` factory or equivalent that holds per-tab book/chapter:

- Create a `createTabState(book, chapter)` function that returns `{ book, chapter }` object
- Add `goNextChapter(tab)` and `goPrevChapter(tab)` that take a tab state
- Keep the global `currentBook`/`currentChapter` as the "active tab's" state for backward compat
- Export a `getChapterForTab(tab)`, `getChapterItemsForTab(tab)` etc. that read from the tab's state instead of globals

### 2. `js/app.js` ‚Äî Tab management and per-tab rendering

**New state:**
- `tabs: Array<{ id, book, chapter, aiResponse, selectedVerses, intent }>``
- `activeTabId` ‚Äî tracks which tab is currently visible
- `nextTabId` ‚Äî auto-incrementing counter

**New UI elements (via JS):**
- Tab bar: `<div id="tab-bar">` inserted above `#split-container` with tab items and an "Add" button
- Each tab shows the passage reference (e.g., "Genesis 1") with a close (X) button
- A `+` button to create a new tab
- Clicking a tab switches to it and updates the navigation bar + verse container
- Minimum 1 tab (last tab is non-closable)

**Key functions:**
- `createTab(book, chapter)` ‚Äî creates a new tab and switches to it
- `closeTab(tabId)` ‚Äî closes a tab, switches to the previous tab if it was the active one
- `switchToTab(tabId)` ‚Äî switches the active tab
- `getActiveTab()` ‚Äî returns the active tab object
- `getSelectedTextForTab(tab)` ‚Äî reads selected verses from the active tab
- `renderTab(tabId)` ‚Äî renders the verse container for the given tab

**Refactored rendering:**
- `renderChapter()` becomes `renderChapterForTab(tab)` ‚Äî reads book/chapter from the tab's state
- `setupVerseSelection(container)` ‚Äî tracks selected verses per tab (from the tab's `selectedVerses` Set)
- Navigation events (book change, chapter change, prev/next) update the active tab's state, then re-render

**AI panel per tab:**
- `sendToAI()` sends for the active tab's selected verses, stores the response in the tab's `aiResponse`
- When switching tabs, restore the AI panel content from the active tab's `aiResponse`
- Clear the `selectedVerses` for the active tab on navigate, restore on switch back

**Persistence:**
- Save tabs state to `localStorage` key `bibleCompanion_tabs` (book/chapter per tab, but not AI responses for performance)
- Restore on load
- On first load with no saved tabs, create one default tab (Genesis 1)

**Keyboard shortcuts:**
- `Ctrl+T` ‚Äî new tab
- `Ctrl+W` ‚Äî close active tab
- `Ctrl+Tab` / `Ctrl+Shift+Tab` ‚Äî cycle tabs (browsers handle this naturally for focus, but we can add numeric shortcuts like `Ctrl+1` through `Ctrl+9`)

### 3. `css/app.css` ‚Äî Tab bar styling

**Tab bar:**
- Inserted between the header and the split container
- Horizontal strip with a background that matches the parchment/parchment-dark theme
- Each tab has a slight 3D bevel effect to look "raised" when active, "sunk" when inactive
- The active tab has a gold accent on the top edge
- Hover effect: gold tint
- Close button (X) appears on hover for each tab
- The `+` add-tab button is styled like the other icon buttons

**Theme variants:**
- Modern theme: rounded tabs, cleaner borders (matching the existing modern theme patterns)

**Tab bar HTML structure (generated by JS):**
```
<div id="tab-bar">
  <div class="tab-item active" data-tab-id="1">
    <span class="tab-title">Genesis 1</span>
    <button class="tab-close" aria-label="Close tab">x</button>
  </div>
  <div class="tab-item" data-tab-id="2">
    <span class="tab-title">John 3</span>
    <button class="tab-close" aria-label="Close tab">x</button>
  </div>
  <button id="add-tab-btn" class="icon-btn" aria-label="New tab">+</button>
</div>
```

### 4. `index.html` ‚Äî Tab bar container

Add `<div id="tab-bar"></div>` between `#app-header` and `#split-container`.

## Flow

1. User opens app ‚Üí default tab created (Genesis 1), verse container + AI panel show for that tab
2. User clicks `+` ‚Üí new tab created with same book/chapter as active tab, verse rendered
3. User navigates in tab ‚Üí changes apply only to active tab
4. User clicks another tab ‚Üí navigation bar updates to that tab's passage, verse container re-renders, AI panel restores that tab's commentary
5. User sends verses to AI ‚Üí commentary is stored in the active tab
6. User closes a tab ‚Üí other tabs unaffected, AI state is lost for the closed tab only
